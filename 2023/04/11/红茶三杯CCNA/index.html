网络结构（冗余，层级）



OSI的概念：

Open System Interconnect开放系统互连参考模型，是由ISO(国际标准化组织)定义的。它是个灵活的、稳健的和可互操作的模型，并不是协议，是用来了解和设计网络体系结构的。

OSI模型的目的：

规范不同系统的互联标准，使两个不同的系统能较容易的通信而不需要改变底层的硬件或软件的逻辑。

02 网络基础 OSI网际互连 下

OSI七层对应功能和作用





应用层提供应用层面的接口，表示层对数据进行编码，会话层建立连接
传输层将数据进行分段，保持端对端连接 网络层将分段以后的报文进行打包，贴上源、目的IP地址 转发数据包，进行路由选择寻址 数据链路层在到达目标网络后进行寻址


三层（网络层）功能及详解

广播、组播控制：

  • 单播 点对点通信
  • 组播 可以控制数据发送范围
  • 广播 限定在广播域(网段)中，其中每一台主机都会收到广播报文

二层详解（数据链路层）

IPv4(点分十进制)为三层地址，亦称逻辑地址
mac(十六进制二十八位)成为二层地址

  • (二层)统一HUB下的所有主机都处于同一个冲突域(collision)中，同时只能有一台设备传输数据，其他设备处于侦听状态，直到链路处于空载状态
  • (二层)交换机每一个端口就是一个独立的冲突域，一个交换机就是一个广播域

例题：主机A telnet连接主机B的过程


1.首先A主机进行数据的封装：

  • 应用层构造数据包，
  • 传输层加上tcp头和源端口(随机)、目的端口号23，
  • 网络层加上IP头，源ip为A的ip，目标为B的ip
  • （未初始化）发送arp请求，获取网关mac地址
  • 数据链路层加上源mac（A）、目标mac（网关的mac）

2.数据以二进制到达路由器：

  • 还原数据帧并提取目的mac地址，确定数据包在局域网的目的是自己
  • 继续还原，提取目的ip地址，查询对应路由
  • 重新封装数据包（目标mac需要通过arp查询获取）

3.数据以二进制到达主机B：

  • 逐层还原，确认目的ip是自己
  • 确认端口并将数据包送到上层对应服务

VLSM

VLSM：可变长子网掩码，全称是Variable Length Subnet Mask。

什么是IP地址

IPv4：三十二位，点分十进制 ：

IPv6 是一个128位的地址空间，通常用8组16进制数表示，每组数之间用冒号分隔。例如：2001:0db8:85a3:0000:0000:8a2e:0370:7334。其中，每组数可以省略前导零，连续的多组 0 可以用双冒号 (::) 来表示一次。“地球上每一粒沙子都能有一个IPv6地址”

十进制数与二进制转换

IP地址的类别


Loopback（本地回环）127.0.0.0/8
A类：2^24 -2 个主机IP 如：10.0.0.0-10.255.255.255
B类：2^16 -2 个主机IP 如：130.130.0.0-130.130.255.255
C类：2^8 -2 个主机IP 如：192.168.2.1-192.168.2.255
路由器寻址只负责将数据包送达目标局域网（根据网络部分）

IP地址的类型


举例： 192.168.2.0/24子网中有主机192.168.2.1，192.168.2为网络号，1为主机号192.168.2.255(255对应二进制11111111)为广播地址

子网掩码


  • 子网号是在划分子网时网络号向主机号借位的二进制数

划分方法： 与网段类型对应， 网络位向主机位借位，从而限制主机位的范围
例如：在上图第一个网段中主机位第一位被限制在 00000000-01111111（对应0-127），
第二个网段主机位第一位被限制在 10000000-11111111（对应128-255）。

示例2： 192.168.3.0/28的子网掩码为255.255.255.240，它表示这个网络可以容纳2^4个主机，即16个主机。其中，4个二进制位被用作主机位，剩下的28-4=24个二进制位被用作网络位。

  • 92.168.3.0 - 192.168.3.15
  • 192.168.3.16 - 192.168.3.31
  • 192.168.3.32 - 192.168.3.47
  • 192.168.3.48 - 192.168.3.63
  • 192.168.3.64 - 192.168.3.79
  • 192.168.3.80 - 192.168.3.95
  • 192.168.3.96 - 192.168.3.111
  • 192.168.3.112 - 192.168.3.127

启用特权执行模式 enable

关闭思科路由器上的域名解析(默认开启)

  1. 进入路由器的配置模式。可以通过在命令行界面中输入 “enable” 命令并输入管理员密码来进入。
  2. 输入 “configure terminal” 命令进入全局配置模式。
  3. 输入 “no ip domain-lookup” 命令以关闭路由器的域名解析功能。
  4. 输入 “exit” 命令返回上一级菜单。
  5. 输入 “write memory” 命令以保存对路由器的更改。
